## 产品概述

"摆地摊了么"是一款地摊分布与发现工具小程序，旨在推广家乡美食文化，方便游客发现地摊、帮助商贩展示信息。

**产品气质**：

> 它不是一个"商家平台"，而是一个"帮人找到摊位的本地指南"。

**核心原则**：

- 名字可以不正规
- 联系可以不完善
- 信息可以不完美
- 但信任会慢慢累积

## 用户注册与身份流转流程

### 用户注册机制（静默注册）

**设计理念**：用户无需感知注册过程，首次使用即自动完成注册

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户首次打开小程序                          │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  app.js onLaunch() 调用 getUserInfo() 云函数                     │
│  - 自动获取微信 OpenID                                           │
│  - 查询 users 集合是否存在该用户                                   │
└─────────────────────────────┬───────────────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              │                               │
              ▼                               ▼
┌─────────────────────────┐      ┌─────────────────────────────┐
│     用户不存在（新用户）   │      │      用户已存在（老用户）      │
│                         │      │                             │
│  自动创建 users 记录：     │      │  直接返回用户信息              │
│  - _openid: 微信OpenID   │      │  - role: user/vendor/admin   │
│  - role: 'user'          │      │  - stallIds: 绑定的摊位        │
│  - nickName: ''          │      │                             │
│  - createTime: 当前时间   │      │                             │
└─────────────────────────┘      └─────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      用户以普通用户身份使用                        │
│              可以：浏览摊位、查看详情、提交反馈                     │
└─────────────────────────────────────────────────────────────────┘
```

**关键实现**：
- 注册时机：小程序 `onLaunch` 时自动调用 `getUserInfo` 云函数
- 注册方式：静默注册，无需用户确认
- 初始角色：所有新用户默认为 `user`（普通用户）
- 数据字段：使用 `_openid`（带下划线）作为查询字段，符合云开发规范

### 用户角色与权限

| 角色 | 标识 | 权限说明 |
|------|------|----------|
| 普通用户 | `user` | 浏览摊位、查看详情、提交反馈 |
| 摊主 | `vendor` | 以上 + 管理自己的摊位、确认摊位状态 |
| 管理员 | `admin` | 以上 + 审核申请、管理所有摊位、进入后台 |

### 数据表关联关系

```
users（用户表）
  ├── _id: 用户唯一ID
  ├── _openid: 微信OpenID
  ├── role: user/vendor/admin
  └── stallIds: [stall._id, ...]  ← 关联

applications（申请表）
  ├── userId: users._id  ← 外键关联
  ├── stallData: { ... }
  └── status: 0待审核/1通过/2拒绝

stalls（摊位表）
  ├── _id: 摊位唯一ID
  ├── ownerUserId: users._id  ← 外键关联
  └── status: 0待审核/1已上架/2已下架
```

### 关键云函数说明

| 云函数 | 用途 | 注册/身份相关逻辑 |
|--------|------|------------------|
| `getUserInfo` | 获取用户信息 | **自动注册新用户**，返回用户角色和绑定摊位 |
| `checkVendorStatus` | 检查摊主状态 | 查询用户角色和申请状态，决定展示哪个页面 |
| `submitApplication` | 提交入驻申请 | 验证用户已注册，创建申请记录 |
| `auditApplication` | 审核申请 | 通过后更新用户 role 为 'vendor' |

## 核心功能（MVP）

- **地图发现**：基于腾讯地图展示地摊分布，支持标记点浏览
- **地摊详情**：展示商品类别、出摊时间、实拍图片、大致位置
- **分类筛选**：按商品类别（烤串/手工/小吃/玩具等）和出摊时段筛选
- **商贩入驻**：支持商贩自主提交信息（需审核）+ 管理员后台录入
- **合规声明**：明确标注"信息展示，不构成交易建议"

## 功能边界

- 不涉及：交易、价格标注、排名、评分、评论区、比价功能
- 不引导：私下交易行为

## 技术栈

- **前端**：微信小程序原生框架（WXML + WXSS + JS）
- **地图服务**：腾讯位置服务 API（微信小程序 map 组件）
- **后端**：微信云开发（CloudBase）
- 云数据库：存储地摊信息、分类数据、审核记录
- 云存储：存储地摊实拍图片
- 云函数：业务逻辑处理、数据校验、审核流程
- **部署**：微信开发者工具 + 云开发控制台

## 开发规范

### 云函数返回格式规范

**统一使用 `code` 字段表示业务状态**：

```javascript
// 成功返回
return {
  code: 0,
  data: { ... },      // 可选，返回的数据
  message: '操作成功'  // 可选，成功提示
};

// 失败返回
return {
  code: -1,
  message: '错误描述'  // 必填，错误信息
};
```

**前端调用示例**：

```javascript
const result = await wx.cloud.callFunction({
  name: 'cloudFunctionName',
  data: { ... }
});

if (result.result.code === 0) {
  // 处理成功逻辑
  const data = result.result.data;
} else {
  // 处理失败逻辑
  throw new Error(result.result.message);
}
```

**注意事项**：
- 不要使用 `success: true/false` 格式（已统一迁移到 `code` 格式）
- 数据库字段初始值不要设为 `null`（会导致后续更新失败），应直接不设置该字段
- 需要动态添加嵌套字段时，使用 `db.command.set()` 替换整个对象

## 设计风格

采用温暖、亲民的设计风格，契合"家乡美食"和"地摊文化"的主题。使用暖色调营造烟火气息，界面简洁明了，降低用户使用门槛。

## 页面规划

### 底部 Tab 结构（4个主入口）

```
📍 附近   |   🧺 逛摊   |   🏮 我是摊主   |   👤 我的
```

**设计原则**：
- 前两个 Tab 给普通用户用
- 后两个 Tab 给"愿意参与的人"用
- 不打扰、不强推，但一直可见
- 摊主入口永远免费、永远可见

---

### ① 📍 附近（地图发现）

**核心目标**：我附近现在有什么摊

**内容结构**（从上到下）：
1. **搜索框**：支持按地摊名称、商品关键词搜索
2. **场景提示文案**：📍 你附近正在摆摊的
3. **时间快捷筛选**：今天 / 晚上 / 周末
4. **摊位列表 / 空状态**：附近地摊卡片列表

**设计要点**：
- 纯用户区，不加任何商贩相关入口
- 地图区域占据主要屏幕，展示地摊标记点
- 底部筛选栏：商品分类快捷筛选、出摊时段筛选
- 定位按钮：快速定位到用户当前位置

---

### ② 🧺 逛摊（分类浏览）

**核心目标**：随便看看，有没有想吃/想逛的

## 风险控制机制

### 一键下线机制

- **摊主自助**：摊主扫码后可随时一键下线自己的摊位
- **管理员**：后台可一键下线任何摊位
- **投诉触发**：收到反馈后，先标记"待确认"并降权展示，不立即删除

### 免责声明

- **页面底部固定显示**："信息展示，不构成交易建议"
- **现场提示**：详情页明确标注"以现场实际为准"
- **入驻协议**：明确告知信息用途，获取口头/线上同意
- **隐私说明**：
- 不收集敏感个人信息
- 位置为大致范围
- 联系方式由摊主主动提供，默认不展示
- **平台定位**：只做信息展示，不做交易中介，不强制联系

### 数据维护策略

| 可信度状态 | 展示策略 | 更新机制 |
| --- | --- | --- |
| 🟢 近期确认（30天内） | 正常展示 | 摊主扫码/管理员确认 |
| 🟡 可能还在（31-90天） | 折叠展示，提示"信息较旧" | 有真实用户访问 |
| 🔴 信息过期（>90天） | 默认不展示，但不删除 | 自动降级 |


### 反馈处理流程

1. 用户点击反馈 → 选择类型（位置不对/商品不符/已不存在）
2. 系统标记摊位为"待确认" → 降权展示
3. 管理员后台查看 → 确认/忽略/下线
4. **原则**：宁可下架，不和人掰扯

### 4. 管理员代申请功能 👨‍💼

**问题**：部分商贩不熟悉小程序操作，无法自主完成入驻申请

**解决方案**：
- **新增摊位字段**：
  ```javascript
  stalls: {
    createdBy: 'admin_proxy', // admin_proxy（管理员代录）/ vendor_self（摊主申请）
    claimStatus: 'unclaimed', // unclaimed（待认领）/ pending（审核中）/ claimed（已认领）/ rejected（审核拒绝）
    claimedBy: '',           // 认领摊主的用户ID
    claimedAt: Date          // 认领时间
  }
  ```
- **管理员代录入流程**：
  1. 管理后台新增"代录入摊位"入口
  2. 表单与普通入驻申请相同（但无需摊主扫码/授权）
  3. 提交后直接上架（`status: 1`）
  4. 摊位状态显示为"营业中（待认领）"
- **摊主认领流程**：
  1. 摊主扫码摊位详情页二维码
  2. 填写认证信息（姓名、手机号、摊位照片等）
  3. 提交审核，等待管理员确认
  4. 审核通过后，摊主获得摊位管理权限

**技术实现**：
- 云函数：`createStallByAdmin`（管理员代录入）
- 云函数：`submitClaimApplication`（提交认领申请）
- 云函数：`auditClaimApplication`（审核认领申请）
- 管理后台页面：新增代录入表单、认领审核列表
- 摊位卡片：待认领状态显示特殊标识

### 6. 入驻申请合规提示 ⚠️

**问题**：部分摊主上传的图片或填写的名称包含价格、促销等交易信息，导致审核不通过

**解决方案**：在申请页面增加友好提示

**提示位置**：

1. **图片上传区域上方**：
```
💡 提示
请上传摊位实拍照片，帮助用户更容易找到你。
照片请不要包含：价格标签、收款码、促销广告。
```

2. **摊位名称输入框下方**：
```
💡 提示
名称用于展示，请不要包含：价格信息、促销用语、联系方式。
例如：不要写"烤串5元"，写"老王烧烤"即可
```

3. **提交前二次确认**：
```
确认信息合规
- 照片无价格、收款码等交易信息 ✓
- 名称无促销、价格等内容 ✓
[确认提交]  [再检查下]
```
