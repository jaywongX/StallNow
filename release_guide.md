## 第六阶段：审核跟进与处理

### 6.1 审核周期

- **一般情况**：1-3个工作日
- **高峰期**（节假日前）：3-7个工作日
- **加急审核**：部分账号有权限，可缩短至1天

### 6.2 常见审核不通过原因及解决

#### 原因1：类目不符

**表现**：
```
你的小程序所选类目与小程序运营内容不符，请补充与小程序内容相关的类目。
```

**解决**：
- 检查小程序实际功能
- 到「设置-基本设置-服务类目」添加正确类目
- 重新提交审核

#### 原因2：涉及UGC但未配置审核机制

**表现**：
```
小程序涉及UGC（用户产生内容），但未配置内容审核机制。
```

**解决**：
- 本小程序已配置申请审核机制，在备注中说明
- 强调所有摊主提交的信息都需要管理员审核后才展示

#### 原因3：隐私协议不完整

**表现**：
```
请完善用户隐私保护指引，明确告知用户收集的信息类型及用途。
```

**解决**：
- 按 2.4.1 配置完整的隐私协议
- 确保 `app.json` 中 `permission` 字段填写完整

#### 原因4：内容不符合规范

**表现**：
```
小程序内容涉及...，不符合《微信小程序平台运营规范》
```

**解决**：
- 检查是否涉及违规内容
- 删除敏感内容后重新提交
- 如有疑问，可提交申诉

### 6.3 审核申诉

**如果认为审核结果有误**：

1. 微信公众平台 → 管理 → 版本管理 → 审核版本
2. 点击「申诉」按钮
3. 填写申诉理由，说明情况
4. 提交申诉

**申诉注意事项**：
- 态度诚恳，说明清楚
- 提供证据截图
- 说明已做的整改

## 第七阶段：正式发布

### 7.1 全量发布

**操作路径**：微信公众平台 → 管理 → 版本管理 → 审核通过 → 发布

**步骤**：

1. 确认审核版本显示「审核通过」
2. 点击「发布」按钮
3. 勾选「已阅读并同意《微信小程序平台服务条款》」
4. 点击「确认发布」

**发布生效时间**：
- 一般即时生效
- 部分用户可能需要5-10分钟才能看到更新

### 7.2 分阶段发布（灰度）

**适用场景**：
- 重大版本更新
- 担心有未发现的bug
- 想逐步放量观察

**操作**：

1. 在发布页面选择「分阶段发布」
2. 选择灰度比例：
   - 第一阶段：10% 用户
   - 第二阶段：50% 用户
   - 第三阶段：100% 用户
3. 观察监控数据，确认无问题后再全量

### 7.3 发布后验证

**验证清单**：

| 验证项 | 操作 | 预期结果 |
|--------|------|----------|
| 搜索小程序 | 在微信搜索小程序名称 | 能找到并打开 |
| 扫码进入 | 扫描小程序码 | 正常进入首页 |
| 分享进入 | 点击分享卡片 | 正常进入对应页面 |
| 核心功能 | 测试地图、列表、详情 | 功能正常 |
| 数据加载 | 检查摊位数据 | 正常显示 |
| 图片显示 | 检查摊位图片 | 正常加载 |

## 第八阶段：发布后维护

### 8.1 数据监控

**监控路径**：微信公众平台 → 统计

**关键指标**：

| 指标 | 关注要点 | 预警值 |
|------|----------|--------|
| 访问人数 | 日活跃用户(DAU) | 持续下降需关注 |
| 访问次数 | 用户活跃度 | 单次使用时长 |
| 新用户数 | 增长趋势 | 推广效果评估 |
| 留存率 | 次日/7日/30日留存 | <20%需优化 |
| 跳出率 | 首页跳出情况 | >50%需优化 |

### 8.2 错误监控

**监控路径**：微信公众平台 → 运维中心 → 错误分析

**关注重点**：
- JavaScript 错误
- API 调用失败
- 页面加载失败
- 云函数错误

### 8.3 用户反馈处理

**收集渠道**：
- 小程序内「反馈」页面
- 微信公众平台 → 客服消息
- 微信小程序评价

**处理流程**：

```
收到反馈 → 分类整理 → 评估优先级 → 安排修复 → 回复用户 → 更新版本
```

### 8.4 版本迭代计划

**迭代周期建议**：
- 小版本（bug修复）：每周
- 中版本（功能优化）：每月
- 大版本（新功能）：每季度

**版本管理**：
- 保留旧版本数据兼容性
- 重大更新需再次审核
- 紧急修复可使用「紧急修复」功能

### 8.5 运营推广建议

**初期推广**：
- 朋友圈分享
- 微信群推广
- 地推（在夜市摆摊点推广）
- 与摊主合作推广

**数据积累**：
- 前期主动录入一些摊位信息
- 邀请摊主入驻
- 鼓励用户反馈新摊位

## 常见问题与解决方案

### Q1: 审核一直不通过怎么办？

**排查步骤**：
1. 仔细阅读拒绝原因
2. 对照《微信小程序平台运营规范》检查
3. 如有疑问，先提交申诉
4. 必要时可咨询微信客服

### Q2: 云开发有免费额度吗？

**免费额度**：
- 数据库读操作：50万次/天
- 数据库写操作：30万次/天
- 云函数调用：500万次/月
- 云存储：5GB空间

**超出后**：按量付费，费用较低

### Q3: 如何设置管理员？

**操作**：
1. 微信公众平台 → 成员管理
2. 添加项目成员
3. 设置权限为「管理员」

### Q4: 小程序名称可以修改吗？

**规则**：
- 个人主体：每年有2次修改机会
- 企业主体：每年有10次修改机会
- 修改后需重新审核

### Q5: 如何生成小程序码？

**操作**：
1. 微信公众平台 → 工具 → 生成小程序码
2. 选择页面路径
3. 下载小程序码

### Q6: 审核期间可以修改代码吗？

**答案**：可以
- 正在审核的版本不会影响
- 可以上传新版本，但会重新排队审核
- 建议等当前审核结果后再决定

### Q7: 发布后发现问题怎么办？

**紧急处理**：
1. 如问题严重，可使用「版本回退」功能
2. 快速修复后重新上传、审核、发布
3. 在小程序内公告告知用户
